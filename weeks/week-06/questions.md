# Вопросы для самопроверки

**1. Зачем нужны variables в GraphQL запросе? Почему нельзя просто склеивать строки?**
Для безопасности и удобства, склейка строк приводит к GraphQL инъекциям и проблемам с экранированием

**2. Из каких полей состоит стандартный JSON-запрос к GraphQL серверу?**
query обязательно и variables - опционально

**3. Почему у GraphQL обычно только один URL endpoint?**
Потому что вся логика запросов описывается в самом запросе, а не в URL

**4. Что означает наличие поля errors в ответе сервера? Может ли оно прийти вместе с data?**
Ошибки в выполнении запроса. Да, может - частичный успех

**5. Как клиент должен обрабатывать ошибки, если HTTP статус ответа 200?**
Проверять наличие поля errors в теле ответа

**6. В чем плюсы использования типизированных клиентов перед простыми HTTP-запросами?**
Автодополнение, валидация на лету, кэширование, типизация данных
