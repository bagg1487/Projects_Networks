## Сравнение gRPC и REST

### REST (HTTP/JSON)
- Всего запросов: 1000
- Общее время: 2.5432 сек
- Среднее время: 2.54 мс
- Запросов в секунду: 393 req/сек

### gRPC (Protocol Buffers)
- Всего запросов: 1000
- Общее время: 1.2145 сек
- Среднее время: 1.21 мс
- Запросов в секунду: 823 req/сек

## Выводы

1. **Производительность**: gRPC оказался в 2.1 раза быстрее REST при одинаковой нагрузке.

2. **Причины**: 
   - Бинарный формат Protocol Buffers компактнее JSON
   - HTTP/2 позволяет мультиплексировать запросы
   - Меньше накладных расходов на парсинг

3. **Streaming преимущества**: gRPC поддерживает стриминг данных, что идеально для real-time приложений (логи, котировки, чаты).

4. **Когда REST лучше**: 
   - Простота отладки
   - Поддержка браузеров
   - Кэширование через CDN
