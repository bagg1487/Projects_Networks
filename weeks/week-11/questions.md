# Вопросы для самопроверки

1. Зачем создавать пользовательскую сеть (bridge network) в Docker Compose, если есть дефолтная?
сервисы видят друг друга по именам (не IP), изоляция от дефолтной сети, DNS-резолвинг
2. В чем разница между `build` и `image` в описании сервиса?
build собирает образ из Dockerfile; image  использует готовый образ из registry.
3. Что делает `depends_on`? Гарантирует ли он, что приложение в зависимом контейнере полностью инициализировалось?
ждёт только запуска контейнера (статус running), НЕ ждёт готовности приложения
4. Как работают healthcheck-и и как использовать их в `depends_on`?
команда проверяет работоспособность (curl/pg_isready). С condition: service_healthy ждёт, пока healthcheck не станет green
5. Как передать переменные окружения в контейнер через Compose (минимум 2 способа)?
через environment: (список), через env_file: (файл .env)
6. Почему Docker Compose редко используют в продакшене (хотя это возможно)?
нет оркестрации (масштабирования, rolling update), восстановления после сбоев, балансировки на нескольких нодах
