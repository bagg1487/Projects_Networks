# Вопросы для самопроверки

1. Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?
    Результат команды. Порядок важен для кэша: изменение команды убивает кэш следующих
2. В чем суть Multi-stage build? Какие проблемы он решает?
    Multi-stage - несколько FROM. Уменьшает размер образа, отделяя сборку от финала.
3. Зачем нужен файл `.dockerignore`?
    .dockerignore исключает лишние файлы из сборки.
4. Чем отличаются инструкции `CMD` и `ENTRYPOINT`?
    CMD - параметры по умолчанию, ENTRYPOINT - фиксированная команда.
5. Почему рекомендуется использовать `COPY` вместо `ADD`?
    COPY - проще и предсказуемее, без магии распаковки.
6. Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?
    0.0.0.0 — доступно снаружи, 127.0.0.1 — только внутри контейнера.
