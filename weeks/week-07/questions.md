# Вопросы для самопроверки

1. Зачем нужен файл `.proto`? Почему мы не пишем классы сообщений сразу на Python?
    Единый контракт для всех языков. Генерация кода вместо ручного написания.
2. Что такое Unary RPC? Чем он концептуально похож на REST запрос?
    Один запрос -> один ответ. Как REST: клиент спрашивает, сервер отвечает.
3. Почему gRPC обычно эффективнее REST (JSON) по процессору и трафику?
    Бинарный формат вместо текста. Быстрее парсится, меньше трафика.
4. Что такое "обратная совместимость" в Protobuf и зачем нужны номера полей (тэги, например `= 1`)?
    Номера полей — это ID, которые не меняются. Можно добавлять новые поля, старые клиенты их игнорируют.
5. Что такое Stub (стаб) на клиенте и Servicer на сервере?
    Stub — клиентская заглушка (вызываешь метод как локальный). Servicer — серверная реализация (наследуешь и пишешь логику).
6. В каких случаях вы бы выбрали REST, а в каких — gRPC?
    REST — для публичных API, браузеров, простоты. gRPC — для внутренних микросервисов, где нужны скорость и строгая типизация.
