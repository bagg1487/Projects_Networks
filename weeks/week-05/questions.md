# Вопросы для самопроверки

1. В чем главное концептуальное отличие GraphQL от REST?
В REST у каждого ресурса свой эндпоинт и сервер решает, какие поля отдавать. В GraphQL один эндпоинт и клиент сам выбирает поля.

2. Что такое Schema Definition Language (SDL)?
Язык описания схемы GraphQL, где определяются типы данных, запросы и мутации.

3. Чем Query отличается от Mutation? Могут ли они выполняться параллельно?
Query - чтение данных, Mutation - изменение. Да, могут выполняться параллельно.

4. Что такое "резолвер" (resolver) и кто его вызывает?
Функция, которая возвращает данные для поля. Вызывается GraphQL движком.

5. Как решается проблема N+1 запросов в GraphQL?
Dataloader'ом - он группирует запросы и выполняет их одним батчем.

6. Почему в GraphQL сложно реализовать кэширование на уровне HTTP?
Потому что все запросы идут на один URL, а тело запроса определяет данные - HTTP кэш по URL не работает.
